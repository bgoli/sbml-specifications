The value of the attribute \token{kind} of a \Unit object must conform to
the syntax of the SBML data type \primtype{UnitSId} and may only take on
the value of a base unit defined in SBML; that is, the value must be one of
the following units:
\val{ampere},
\val{avogadro},
\val{becquerel},
\val{candela},
\val{coulomb},
\val{dimensionless},
\val{farad},
\val{gram},
\val{gray},
\val{henry},
\val{hertz},
\val{item},
\val{joule},
\val{katal},
\val{kelvin},
\val{kilogram},
\val{litre},
\val{lumen},
\val{lux},
\val{metre},
\val{mole},
\val{newton},
\val{ohm},
\val{pascal},
\val{radian},
\val{second},
\val{siemens},
\val{sievert},
\val{steradian},
\val{tesla},
\val{volt},
\val{watt}, or
\val{weber}.
The SBML unit system is not hierarchical, and user-defined units cannot be
defined using other user-defined units.  (References: SBML L3V1 Section 4.4.2; SBML L3V2
\sec{sec:unit-structure}.)
