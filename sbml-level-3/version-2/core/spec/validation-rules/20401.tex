The value of the attribute \token{id} in a \UnitDefinition object
must conform to the syntax of the SBML data type
\primtype{UnitSId} and not be identical to any unit predefined in SBML.
That is, the identifier must not be the same as any of the following base
units:
\val{ampere},
\val{avogadro},
\val{becquerel},
\val{candela},
\val{coulomb},
\val{dimensionless},
\val{farad},
\val{gram},
\val{gray},
\val{henry},
\val{hertz},
\val{item},
\val{joule},
\val{katal},
\val{kelvin},
\val{kilogram},
\val{litre},
\val{lumen},
\val{lux},
\val{metre},
\val{mole},
\val{newton},
\val{ohm},
\val{pascal},
\val{radian},
\val{second},
\val{siemens},
\val{sievert},
\val{steradian},
\val{tesla},
\val{volt},
\val{watt}, or
\val{weber}.
(References: SBML L3V1 Section~\ref{sec:unit-structure}.)
