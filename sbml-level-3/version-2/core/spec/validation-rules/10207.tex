The only permitted values for the attribute \token{type} on MathML
\token{cn} elements are \val{e-notation}, \val{real}, \val{integer}, and
\val{rational}.
\changed{An SBML package may allow new values for the \token{type} attribute, and if so, the package must define \token{required}=\val{true} on the SBML container element \token{<sbml>}.}
(References: \tworefs{3.4.2}{\ref{sec:cn-token}}.)
