The value of the \token{id} field in a \UnitDefinition must not be
identical to any unit predefined in SBML.  That is, the identifier
must not be the same as a value from the \primtype{UnitKind}
enumeration (i.e.,
\val{ampere}
\val{gram}
\val{katal}
\val{metre}
\val{second}
\val{watt}
\val{becquerel}
\val{gray}
\val{kelvin}
\val{mole}
\val{siemens}
\val{weber}
\val{candela}
\val{henry}
\val{kilogram}
\val{newton}
\val{sievert}
\val{coulomb}
\val{hertz}
\val{litre}
\val{ohm}
\val{steradian}
\val{dimensionless}
\val{item}
\val{lumen}
\val{pascal}
\val{tesla}
\val{farad}
\val{joule}
\val{lux}
\val{radian}
\val{volt}.)
(References: L2V1 erratum 14; L2V2
Section~\ref{sec:unit-structure}.)
