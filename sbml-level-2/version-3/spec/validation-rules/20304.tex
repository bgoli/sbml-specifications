Inside the \token{lambda} of a \FunctionDefinition, if a
\token{ci} element is not the first element within a MathML
\token{apply}, then the \token{ci}'s value can only be the value
of a \token{bvar} element declared in that \token{lambda}.  In
other words, all model entities referenced inside a function
definition must be passed as arguments to that function.
(References: L2V2
L2V2 Section~3.5.3 and~4.3.2; L2V3 Sections~\ref{sec:ci-token}
and~\ref{sec:function-definition-math}.)
