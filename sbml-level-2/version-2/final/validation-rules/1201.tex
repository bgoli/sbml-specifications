The value of the \token{id} field in a \UnitDefinition must not be
identical to any unit predefined in SBML.  That is, the identifier must not
be the same as a value from the \primtype{UnitKind} enumeration (e.g.,
\val{litre}, \val{mole}, \val{metre}, etc.).  (References: L2V1
erratum 14; L2V2 Section~\ref{sec:unit-structure}.)
